Dependency Injection

What's DI?

Dependency Injection (DI) is a software design pattern that deals with how components get hold of their dependencies. The Angular injector subsystem is in charge of creating components, resolving their dependencies, and providing them to other components as requested.

Why we need DI into our application?

It's useful to have each component declaring its dependencies and then get them injected into it by Angular.

It's possible to inject constants, values, functions and objects.

Best practices

Controllers, services, directives, filters and animations can be injected with value and service components.

angular.run method can be injected with service, value and constant components.

angular.config method can be injected with provider and constant.

Vendor/Personal dependencies

AngularJS lets you to inject various types of dependencies:

  • Angular built-in dependencies
  • Vendor dependencies
  • Personal dependencies

Code sample


angular.run(['$http', function ($http) {
    // do something with $http service
}]);
angular.config(['$httpProvider', function ($httpProvider) {
    // do something with $httpProvider provider
}]);
angular.controller('MyCtrl', ['$scope', function ($scope) {
    // do something with $scope service
}]);

Template Engine

Why Angular has a template engine?

RESTful single page web applications need to compile and render their markup. This means that only raw data is sent/received to/from the network.

How does it work?

Angular combines the template with information from the model and controller to render the dynamic view that a user sees in the browser.

The $compile service is used to compile the DOM and link it with $rootScope

Compile Phase

The compiler ($compile) traverse the DOM and collect all of the directives. The result is a linking function

Link Phase

Combine the directives with a scope and produce a live view. Any changes in the scope model are reflected in the view, and any user interactions with the view are reflected in the scope model. This makes the scope model the single source of truth

Template Composition

The template engine supports the following AngularJS elements and attributes:

  • Directive
  • Expression
  • Filter
  • Form controls

Views

It's possible to split in different views, called partials, the Angular markup. Those views are kept in HTML files.

What is a directive?

Directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler ($compile) to attach a specified behavior to that DOM element or even transform the DOM element and its children.

Build your custom directives

One of the most powerful feature in AngularJS is the possibility to make custom directives. Like other Angular components, a custom directive is just a module.

Configuration

A directive must return a configuration object with the following options:

  • priority
  • require
  • restrict
  • transclude
  • scope
  • controller
  • controllerAs
  • link
  • templateNamespace
  • templateUrl
  • template

The link function

The link function is responsible for registering DOM listeners as well as updating the DOM. It is executed after the template has been cloned.

It comes with the following signature:


function link(scope, iElement, iAttrs, controller, transcludeFn) { ... }

Scope in directives

If set to true, then a new scope will be created for this directive. If multiple directives on the same element request a new scope, only one new scope is created. The new scope rule does not apply for the root of the template since the root of the template always gets a new scope.

If set to {} (object hash), then a new "isolate" scope is created. The 'isolate' scope differs from normal scope in that it does not prototypically inherit from the parent scope. This is useful when creating reusable components, which should not accidentally read or modify data in the parent scope.

Controller integration

A directive can have its own custom controller (like the scope) and it's possible to share the logic between other directives.

A controller is injectable with the following locals:

  • $scope
  • $element
  • $attrs
  • $transclude

Transclusion

Transclusion is the process of extracting a collection of DOM element from one part of the DOM and copying them to another part of the DOM, while maintaining their connection to the original AngularJS scope from where they were taken.

Transclusion is used (often with ngTransclude) to insert the original contents of a directive's element into a specified place in the template of the directive. The benefit of transclusion, over simply moving the DOM elements manually, is that the transcluded content has access to the properties on the scope from which it was taken, even if the directive has isolated scope.

This makes it possible for the widget to have private state for its template, while the transcluded content has access to its originating scope.

Filters

What is a filter?

A filter formats the value of an expression for display to the user. They can be used in view templates, controllers or services and it is easy to define your own filter.

Built-in filters

AngularJS has several built-in filters:

  • currency
  • date
  • json
  • limitTo
  • lowercase
  • uppercase
  • number
  • orderBy

// TODO: add link to filter docs

Filters within templates

Filters can be applied to expressions in view templates using the following syntax:


{{ expression | filter }}

It's possible to chain more filters:


{{ expression | filter1 | filter2 | ... }}

Filters may have arguments:


{{ expression | filter:argument1:argument2:... }}

Filters within the application

Inject a dependency with the name <filterName>Filter to your controller/service/directive:


angular.controller('MyCtrl', ['dateFilter', function(dateFilter) {
    var date = dateFilter(new Date(), 'Ymd');
}]);

Define your custom filters

Writing your own filter is very easy: just register a new filter factory function with your module. This factory function should return a new filter function which takes the input value as the first argument. Any filter arguments are passed in as additional arguments to the filter function.

Code sample


angular.filter('star', function(uppercaseFilter) {
    return function (value, starNumber) {
        value = String(value);
        starNumber = starNumber || 0;
        var stars = '';

        for (var i = 0; i < starNumber; i++) stars += '*';
        result.push(stars);
        result.push(uppercaseFilter(value));
        result.push(stars);
        if (value && value.length > 0) return result.join(' ');
        else return '';
    };
});

Services | Factories | Providers

What's a service?

Angular services are substitutable objects that are wired together using dependency injection (DI). You can use services to organize and share code across your app.

Angular services are:

  • Lazily instantiated – Angular only instantiates a service when an application component depends on it.
  • Singletons – Each component dependent on a service gets a reference to the single instance generated by the service factory.

Angular offers several useful services (like $http), but for most applications you'll also want to create your own.

Using a service

Angular has several built-in services and it's possible to use them with the Dependency Injection, like so: 


angular.controller('MyCtrl', ['$http', function ($http) {
    $http.get('/user')
         .success(function (user) {
            // Use the user data
         })
         .error(function (err) {
            // Notify the error
         });
}]);

A service can be used where it's possible to use the DI: in other services, directives, controllers and filters.

Built-in services

AngularJS has several built-in services:

Service, factory and provider

There are several ways to share code across the application:

  • provider(provider) - registers a service provider with the $injector
  • service(class): when declaring a service as an injectable argument you will be provided with an instance of the class. In other words new ClassYouPassedToService()
  • factory(fn): when declaring a factory as an injectable argument you will be provided with the value that is returned by invoking the function reference passed to module.factory

Value and constant

Another way to share code is with angular.value and angular.constant:

  • constant(obj) - registers a value/object that can be accessed by providers and services.
  • value(obj) - registers a value/object that can only be accessed by services, not providers.

Data Validation

Angular form

Form is simply a group of controls.

Angular form creates an instance of FormController. FormController itself has methods and properties:

  • formName.$pristine: TRUE if the user has not interacted with the form yet
  • formName.$dirty: TRUE if the user has already interacted with the form.
  • formName.$valid: TRUE if all containing form and controls are valid
  • formName.$invalid: TRUE if at least one containing form and control is invalid.
  • formName.$error: Is an object hash, containing references to all invalid controls or forms

Form states

It is important to understand flow of the form states in order to use angular form properly. This flow gives you visualization of the form state from the very first time the form is rendered until the user has finished filling the form

  • Flow 1: pristine and invalid - When the form is first rendered and the user has not interacted with the form yet.
  • Flow 2: dirty and invalid - User has interacted with the form, but validity has not been satisfied, yet
  • Flow 3: dirty and valid - User has finished filling the form and all the validation rule has been satisfied

Data validation and data binding

Thanks to the ngModel is possible to combine the data-binding with the data validation. The ngModelController has several validation functions stored in the $validators object and Angular provides basic implementation for most common HTML5 input types: (text, number, url, email, radio, checkbox), as well as some directives for validation (required, pattern, minlength, maxlength, min, max).

With directives it's possible to add new type of validators: each function in the $validators object receives the modelValue and the viewValue as parameters that Angular uses to validate the form.

Routing

Client-side routing

Single page web applications need client-side routing because they communicate with the server with AJAX/WebSocket protocols instead of refreshing the entire page.

Angular provides a set of spectacular features.

Routing w/ templating

The ngRoute module provides routing and deeplinking services and directives for angular apps. The $routeProvider is used to configure the routing table: each view can be attached to a route, combined with the ngView directive.

Code sample


angular.module('MyModule', ['ngRoute'])
    .config(['$routeProvider', function ($routeProvider) {
        $routeProvider.when('/', {
            templateUrl: 'views/mario.html',
            controller: 'MarioCtrl'
        })
        .otherwise ({
            redirectTo: '/404'
        });
    }]);

Testing

Types of testing

First of all, Javascript is a dynamically typed language that comes with no help from the compiler and Angular is written with testability in mind. There are two types of testing in AngularJS:

  • End to End Testing
  • Unit Testing

E2E or Unit Test?

As applications grow in size and complexity, it becomes unrealistic to rely on manual testing to verify the correctness of new features, catch bugs and notice regressions. Unit tests are the first line of defense for catching bugs, but sometimes issues come up with integration between components which can't be captured in a unit test. End to end tests are made to find these problems.

Jasmine (Unit Test)

Jasmine is a testing framework and has several powerful constructors that allow to make assertions and expections on different expression.


var Mario;
describe('Testing Mario service', function () {
    beforeEach(module('MyModule'));
    beforeEach(inject(function (_Mario_) {
        Mario = _Mario_;
    }));

    describe('Testing getCoins method', function () {
        it('should return an empty array of coins', function () {
            var coins = Mario.getCoins();
            expect(coins).toBeDefined();
            expect(coins.length).toBeDefined();
            expect(coins.length).toEqual(0);
        });
    });
});

Protractor (E2E Test)

Protractor is an end to end test runner which simulates user interactions that will help you verify the health of your Angular application.


describe('Users Table', function () {
    it('should filter the users by name', function () {
        // Load users
        element(by.css(':button')).click();
        expect(element.all(by.repeater('user in users')).count()).toEqual(3);

        // Filter by name (Mario)
        element(by.model('name')).sendKeys('Mario');
        expect(element.all(by.repeater('user in users')).count()).toEqual(1);
    });
});

Useful Modules

ngResource

What is it?

The ngResource module provides interaction support with RESTful services via the $resource service and $resourceProvider.

$resource service

A factory which creates a resource object that lets you interact with RESTful server-side data sources.

When a new $resource is created, it's expected to pass the following parameters to the constructor:

  • (String) url: a string representing the url of this resource
  • (Object) paramDefaults: a key-value map representing default parameters that will be attached to each call in the query string
  • (Object) actions: Hash with declaration of custom action. Each action will be reflected as a resource method and it will have its custom HTTP method, url and params.

Code sample


angular.module('MyModule', ['ngResource'])
    .service('Mario', ['$resource', function ($resource) {
        return $resource('/Mario/:id', {id: '@id'}, {
            getStars: {
                method: 'GET',
                url: '/Mario/stars',
                isArray: true
            }
        });
    }])
    .controller('MyCtrl', ['$scope', 'Mario', function ($scope, Mario) {
        // -> GET /Mario/
        // <- {id: 10, starCounter: 9}
        $scope.mario = Mario.get(function () {
            $scope.mario.starCounter = 10;
            $scope.mario.$save(); // POST /Mario/10 {id: 10, starCounter: 10}
        });
        // GET /Mario/stars
        $scope.stars = Mario.getStars(function () {
            expect($scope.stars.length).not.toEqual(0);
        });
    }]);

ngRoute

What is it?

ngRoute is an Angular module providing features to handle client-side routing with ease. It's mainly composed of:

  • ngView: directive
  • $routeProvider: provider
  • $route: service
  • $routeParams: service

The idea is to use the $routeProvider initially, combined with the ngView directive to load the route template associated and then to use the $route/$routeParams service to get params.

$routeProvider

This provider is used to configure the routing table of your application. It has two methods: when and otherwise. For each route defined with when method it's possible to configure the following properties:

  • controller: the controller that will handle the route
  • template: an Angular template string
  • templateUrl: the url of a view (a template located in a file)
  • resolve: An optional map of dependencies which should be injected into the controller.
  • redirectTo: a string defining the route to redirect

$routeParams

The $routeParams service allows you to retrieve the current set of route parameters.


angular.controller('MyCtrl', ['$scope', '$routeParams', function ($scope, $routeParams) {
    // localhost/#/wilk/ngWebsocket/ <- mapped with $routeProvider
    $scope.username = $routeParams.username;
    $scope.reponame = $routeParams.reponame;
}]);

Code sample


angular.module('MyModule', ['ngRoute'])
    .config(['$routeProvider', function ($routeProvider) {
        $routeProvider.when('/:username/:reponame/', {
            templateUrl: 'views/repo.html',
            controller: 'RepoCtrl'
        });
    }])
    .controller('MyCtrl', ['$scope', '$routeParams', function ($scope, $routeParams) {
        $scope.username = $routeParams.username;
        $scope.reponame = $routeParams.reponame;
    }]);

ngCookie

What is it?

The ngCookies module provides a convenient wrapper for reading and writing browser cookies.

It comes with two services:

  • $cookieStore
  • $cookies

$cookieStore

Provides a key-value (string-object) storage, that is backed by session cookies. Objects put or retrieved from this storage are automatically serialized or deserialized by angular's toJson/fromJson.


angular.module('MyModule', ['ngCookies'])
    .controller('MyCtrl', ['$cookieStore', function ($cookieStore) {
        $cookieStore.put('Character', 'Mario');
        var choosenCharacter = $cookieStore.get('Character'); // Mario
        $cookieStore.remove('Character');
    }]);

$cookies

Provides read/write access to browser's cookies like $cookieStore. The mainly difference is that only a simple object is exposed and by adding or removing properties to/from this object, new cookies are created/deleted.


angular.module('MyModule', ['ngCookies'])
    .controller('MyCtrl', ['$cookies', function ($cookies) {
        var Character = $cookies.Character;
        $cookies.Character = 'Mario';
    }]);
