Dependency Injection

What's DI?

Dependency Injection (DI) is a software design pattern that deals with how components get hold of their dependencies. The Angular injector subsystem is in charge of creating components, resolving their dependencies, and providing them to other components as requested.

Why we need DI into our application?

It's useful to have each component declaring its dependencies and then get them injected into it by Angular.

It's possible to inject constants, values, functions and objects.

Best practices

Controllers, services, directives, filters and animations can be injected with value and service components.

angular.run method can be injected with service, value and constant components.

angular.config method can be injected with provider and constant.

Vendor/Personal dependencies

AngularJS lets you to inject various types of dependencies:

  • Angular built-in dependencies
  • Vendor dependencies
  • Personal dependencies

Code sample


angular.run(['$http', function ($http) {
    // do something with $http service
}]);
angular.config(['$httpProvider', function ($httpProvider) {
    // do something with $httpProvider provider
}]);
angular.controller('MyCtrl', ['$scope', function ($scope) {
    // do something with $scope service
}]);

Template Engine

Why Angular has a template engine?

RESTful single page web applications need to compile and render their markup. This means that only raw data is sent/received to/from the network.

How does it work?

Angular combines the template with information from the model and controller to render the dynamic view that a user sees in the browser.

The $compile service is used to compile the DOM and link it with $rootScope

Compile Phase

The compiler ($compile) traverse the DOM and collect all of the directives. The result is a linking function

Link Phase

Combine the directives with a scope and produce a live view. Any changes in the scope model are reflected in the view, and any user interactions with the view are reflected in the scope model. This makes the scope model the single source of truth

Template Composition

The template engine supports the following AngularJS elements and attributes:

  • Directive
  • Expression
  • Filter
  • Form controls

Views

It's possible to split in different views, called partials, the Angular markup. Those views are kept in HTML files.

Directives

What is a directive?

Directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler ($compile) to attach a specified behavior to that DOM element or even transform the DOM element and its children.

Built-in directives

Angular comes with several built-in directives:

  • ngApp
  • ngController
  • ngModel
  • ngRepeat
  • ngClick
  • ngView

Built-in directives are identified by the ng namespace.

Build your custom directives

One of the most powerful feature in AngularJS is the possibility to make custom directives. Like other Angular components, a custom directive is just a module.

Configuration

A directive must return a configuration object with the following options:

  • priority
  • require
  • restrict
  • transclude
  • scope
  • controller
  • controllerAs
  • link
  • templateNamespace
  • templateUrl
  • template

The link function

The link function is responsible for registering DOM listeners as well as updating the DOM. It is executed after the template has been cloned.

It comes with the following signature:


function link(scope, iElement, iAttrs, controller, transcludeFn) { ... }

Scope in directives

If set to true, then a new scope will be created for this directive. If multiple directives on the same element request a new scope, only one new scope is created. The new scope rule does not apply for the root of the template since the root of the template always gets a new scope.

If set to {} (object hash), then a new "isolate" scope is created. The 'isolate' scope differs from normal scope in that it does not prototypically inherit from the parent scope. This is useful when creating reusable components, which should not accidentally read or modify data in the parent scope.

Controller integration

A directive can have its own custom controller (like the scope) and it's possible to share the logic between other directives.

A controller is injectable with the following locals:

  • $scope
  • $element
  • $attrs
  • $transclude

Transclusion

Transclusion is the process of extracting a collection of DOM element from one part of the DOM and copying them to another part of the DOM, while maintaining their connection to the original AngularJS scope from where they were taken.

Transclusion is used (often with ngTransclude) to insert the original contents of a directive's element into a specified place in the template of the directive. The benefit of transclusion, over simply moving the DOM elements manually, is that the transcluded content has access to the properties on the scope from which it was taken, even if the directive has isolated scope.

This makes it possible for the widget to have private state for its template, while the transcluded content has access to its originating scope.

Filters

What is a filter?

A filter formats the value of an expression for display to the user. They can be used in view templates, controllers or services and it is easy to define your own filter.

Built-in filters

AngularJS has several built-in filters:

  • currency
  • date
  • json
  • limitTo
  • lowercase
  • uppercase
  • number
  • orderBy

Filters within templates

Filters can be applied to expressions in view templates using the following syntax:


{{ expression | filter }}

It's possible to chain more filters:


{{ expression | filter1 | filter2 | ... }}

Filters may have arguments:


{{ expression | filter:argument1:argument2:... }}

Filters within the application

Inject a dependency with the name <filterName>Filter to your controller/service/directive:


angular.controller('MyCtrl', ['dateFilter', function(dateFilter) {
    var date = dateFilter(new Date(), 'Ymd');
}]);

Define your custom filters

Writing your own filter is very easy: just register a new filter factory function with your module. This factory function should return a new filter function which takes the input value as the first argument. Any filter arguments are passed in as additional arguments to the filter function. 


angular.filter('MyFilter', function(dateFilter) {
    return function (value, secondArg) {
        return '*** ' + secondArg + value + secondArg + ' ***';
    };
});

Services | Factories | Providers

What's a service?

Angular services are substitutable objects that are wired together using dependency injection (DI). You can use services to organize and share code across your app.

Angular services are:

  • Lazily instantiated – Angular only instantiates a service when an application component depends on it.
  • Singletons – Each component dependent on a service gets a reference to the single instance generated by the service factory.

Angular offers several useful services (like $http), but for most applications you'll also want to create your own.

Using a service

Angular has several built-in services and it's possible to use them with the Dependency Injection, like so: 


angular.controller('MyCtrl', ['$http', function ($http) {
    $http.get('/user')
         .success(function (user) {
            // Use the user data
         })
         .error(function (err) {
            // Notify the error
         });
}]);

A service can be used where it's possible to use the DI: in other services, directives, controllers and filters.

Creating a custom service

angular.service allows to define a new service: 


angular.service('Mario', ['$http', function ($http) {
    this.salutation = function () {
        console.log('Hello, I\'m Mario!');
    };

    this.followPath = function (path) {
        $http.get(path).success(onSuccess).error(onError);
    };
}]);

Service, factory, value, provider, ...?

There are several ways to share code across the application:

  • Service: when declaring a service as an injectable argument you will be provided with an instance of the function. In other words new FunctionYouPassedToService()
  • Factory: when declaring a factory as an injectable argument you will be provided with the value that is returned by invoking the function reference passed to module.factory
  • Value: it returns just a value
  • Provider: it allows to configure a service during the module configuration phase
